# Transport Adjacency Matrix

Legend: `covered` means the current contract has an explicit schema/profile; `partial` means related fields exist but the concern is not fully modeled; `missing` means a new schema/profile is needed.

| Concern | Layer | Current support | Scope impact | Event / frame impact | Proposed schema(s) | Primary fields to track | Validation / policy notes |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Trailers | ASGI HTTP event | missing | `scope.type=http`; applies to `rest`, `jsonrpc`, `http.stream`, `sse` | Add `http.response.trailers`; optionally request trailer profile if supported later | `contract/schemas/events/http-response-trailers.schema.json`, `contract/schemas/headers.schema.json` | `headers`, `more_trailers`; `http.response.start.trailers=true` prerequisite | Response trailers are valid only after body frames when trailers were announced. Header pair encoding must match ASGI byte-pair policy. |
| Early hints | HTTP informational response | missing | `scope.type=http`; mostly `https`/H2/H3 useful but not limited to them | Add informational response frame/profile for `103` before final response | `contract/schemas/events/http-response-informational.schema.json` | `status=103`, `headers`, `links` derived from `Link` headers | Must not replace final `http.response.start`; should be constrained to 1xx status range, especially `103`. |
| Redirect | HTTP response behavior | partial | `scope.type=http`; effective `scheme`, `host`, `root_path` matter behind proxy | Uses normal `http.response.start` and `http.response.body` | `contract/schemas/profiles/redirect.schema.json` | `status`, `location`, `absolute`, `relative`, `trusted_forwarded_inputs` | Redirect correctness depends on proxy forwarding trust, effective scheme/host/port/root path, and allowed status codes. |
| Upgrade | HTTP protocol transition | partial | Starts from `scope.type=http`; may transition to `websocket` or tunnel/profile | WebSocket upgrade represented as `websocket.connect`; generic HTTP Upgrade and CONNECT are not explicit | `contract/schemas/profiles/upgrade.schema.json`, `contract/schemas/events/http-upgrade.schema.json` | `upgrade_protocol`, `connection`, `sec_websocket_*`, `connect_protocol`, `accepted` | WebSocket is standard ASGI scope behavior; non-WebSocket upgrade and extended CONNECT need explicit profile boundaries. |
| Proxy forwarding | Deployment metadata | missing | Affects effective `client`, `server`, `scheme`, `root_path`, `headers`, `secure` | No direct ASGI event; influences request interpretation and generated redirects/URLs | `contract/schemas/ext/forwarded.schema.json` | `trusted`, `trusted_hops`, `trusted_proxies`, `source`, `for`, `proto`, `host`, `port`, `prefix`, `raw`, effective values | Untrusted forwarded headers must be ignored or preserved-only. Must distinguish `Forwarded`, `X-Forwarded-*`, `X-Real-IP`, CDN headers, and proxy protocol. |
| TLS | ASGI extension / transport security | partial | Applies to TLS-bearing `http` and `websocket` scopes; informs `secure`, `scheme`, `alpn` | No separate events; ASGI TLS extension says base protocol events remain unchanged | `contract/schemas/ext/tls.schema.json` | `server_cert`, `client_cert_chain`, `client_cert_name`, `client_cert_error`, `tls_version`, `cipher_suite` | Align with ASGI `scope.extensions.tls`; current `ext.transport.tls` is useful but not shape-compatible. |
| mTLS | TLS client-auth profile | partial | Applies when TLS client certificate is requested or required | No separate events; affects identity/authorization metadata | `contract/schemas/ext/mtls.schema.json` | `required`, `verified`, `client_cert_chain`, `client_cert_name`, `client_cert_error`, `peer_identity`, `trust_store_id` | Must distinguish absent cert, present verified cert, present failed cert, and cert forwarded by trusted proxy/load balancer. |
| SVCB | DNS service binding discovery | missing | Pre-scope discovery; may influence selected endpoint, ALPN, port, ECH, address hints | No ASGI event; discovery metadata before connection | `contract/schemas/ext/svcb.schema.json` | `owner_name`, `svc_priority`, `target_name`, `svc_params`, `alpn`, `port`, `ipv4hint`, `ipv6hint`, `ech` | Track as discovery metadata, not as an ASGI scope type. Should support alias mode and service mode. |
| HTTPS scheme | ASGI HTTP scheme / secure transport | partial | `scope.type=http`, `scheme=https`, `ext.transport.secure=true`, usually TLS present | Uses normal HTTP events | `contract/schemas/scheme.schema.json`, `contract/schemas/ext/tls.schema.json` | `scheme`, `secure`, `tls`, `forwarded.proto`, `effective_scheme` | `https` is not a separate scope. Behind proxy, effective scheme must come from trusted forwarding or direct TLS. |
| HTTPS DNS record | DNS HTTPS RR discovery | missing | Pre-scope discovery; specialized SVCB for HTTPS origins | No ASGI event; discovery metadata before connection | `contract/schemas/ext/https-rr.schema.json` | `name`, `svc_priority`, `target_name`, `svc_params`, `alpn`, `port`, `ech`, `ipv4hint`, `ipv6hint` | Keep distinct from URI scheme `https`. This is a DNS RR type using SVCB parameter semantics. |
| Alt-Svc | HTTP alternative service advertisement | missing | Response/header metadata can influence future connection scope selection | Response header carried in `http.response.start`; should also be parsed as service advertisement | `contract/schemas/ext/alt-svc.schema.json` | `protocol_id`, `authority`, `host`, `port`, `ma`, `persist`, `clear`, `raw` | Should validate advertised protocols such as `h2` and `h3`; does not change the current request scope, only future alternatives. |
| ALPN | TLS protocol negotiation | partial | `ext.transport.alpn`; constrains HTTP/WebSocket profile such as `http/1.1`, `h2`, `h3` | No separate event; determines protocol mapping and available frames | `contract/schemas/alpn.schema.json` | `offered`, `selected`, `source`, `protocol_id` | Enumerate known values while permitting extension values. Must align with TLS and SVCB/Alt-Svc advertisements. |
| ECH | TLS privacy / discovery parameter | missing | Pre-connection discovery plus TLS handshake metadata | No ASGI event; affects TLS establishment metadata | `contract/schemas/ext/ech.schema.json` | `advertised`, `config_list`, `accepted`, `public_name`, `source` | Usually advertised through HTTPS/SVCB. Track without requiring applications to parse raw ECH config unless available. |
| HTTP/3 advertisement | Discovery and response advertisement | missing | Pre-scope or response-derived; may lead to future `network=quic`, `alpn=h3` | `Alt-Svc` header and/or HTTPS/SVCB record; H3 frame profile after negotiation | `contract/schemas/profiles/http3-advertisement.schema.json`, `contract/schemas/ext/alt-svc.schema.json`, `contract/schemas/ext/https-rr.schema.json` | `source`, `h3_alpn`, `authority`, `port`, `max_age`, `persist`, `svc_params` | Advertisement is not the same as negotiated H3. Need separate fields for advertised, selected, and effective protocol. |
| HTTPS/SVCB | Combined DNS discovery profile | missing | Pre-scope discovery profile for HTTPS service binding records and generic SVCB | No ASGI event; influences endpoint and ALPN selection before connection | `contract/schemas/ext/discovery.schema.json`, `contract/schemas/ext/svcb.schema.json`, `contract/schemas/ext/https-rr.schema.json` | `rr_type`, `priority`, `target`, `params`, `alpn`, `port`, `ech`, `address_hints` | Keep combined profile for policy/reporting, but retain separate schemas for generic SVCB and HTTPS RR. |
